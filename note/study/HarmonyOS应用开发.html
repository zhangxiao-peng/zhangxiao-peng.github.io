HarmonyOS应用开发

应用程序入口 UIAbility

包含用户界面的应用组件 用于用户交互 是系统调度单元
一个应用可以有一个也可以有多个 

生命周期
  onCreate  实例创建时触发 进行初始化
  onWindowStageCreate 设置UI页面加载、设置WindowStage的事件订阅,通过loadContent接口设置应用要加载的页面
  onForeground 切换前台 申请系统需要的资源，或者重新申请在onBackground中释放的资源。
  onBackground 切换后台 释放UI页面不可见时无用的资源，或者在此回调中执行较为耗时的操作
  onWindowStageDestroy 释放UI页面资源
  Destroy 进行系统资源的释放、数据的保存等操作。
  
  每一个UIAbility实例都对应持有一个WindowStage实例 本地窗口管理器（界面聚焦/失焦/可见/不可见）
  
页面跳转和数据传递 
    通过页面路由模块实现访问不同页面
    import router from '@ohos.router';//导入router模块
    
    router.pushUrl({url: 'pages/Second',params: {}}, router.RouterMode.Single)
    router.getParams(); //方法获取页面传参
    router.back(); //返回上一个页面 返回原来的页面不会重新声明变量 也不会触发页面的aboutToAppear回调
    router.back({ url: 'pages/Index',params:{} });//返回指定页面
    router.enableBackPageAlert();//方法开启页面返回询问对话框功能
    router.disableAlertBeforeBackPage();//关闭页面返回询问对话框
页面跳转方式
router.pushUrl()router.replaceUrl()
mode参数配置为router.RouterMode.Single 单实例模式 router.RouterMode.Standard 多实例模式mode参数配置为router.RouterMode.Single 单实例模式 router.RouterMode.Standard 多实例模式
目标页面的url在页面栈存在同url 离栈顶最近同url会被移动到栈顶,移动后的页面为新建页，原来的页面仍然存在栈中，页面栈的元素数量不变；如果目标页面的url在页面栈中不存在同url页面，按多实例模式跳转，页面栈的元素数量会加1。如果目标页面的url在页面栈中已经存在同url页面，离栈顶最近同url页面会被移动到栈顶，替换当前页面，并销毁被替换的当前页面，移动后的页面为新建页，页面栈的元素数量会减1；如果目标页面的url在页面栈中不存在同url页面，按多实例模式跳转，页面栈的元素数量不变
当页面栈的元素数量较大或者超过32时，可以通过调用router.clear()方法清除页面栈中的所有历史页面，仅保留当前页面作为栈顶页面

UIAbility的启动模式：

module.json5文件中的“launchType”字段配置

单实例模式 singleton （默认）多实例模式 multiton指定实例模式 specified
startAbility()方法调用时 进程中实例存在 则复用(只会进入onNewWant回调) 系统存在唯一该UIAbility实例startAbility()方法调用时 ，都会新建 最近任务列表会有多个该类型的UIAbility实例针对特殊场景 在启动指定实例模式的UIAbility时，给每一个UIAbility实例配置一个独立的Key标识，want参数中增加一个自定义参数，调用startAbility()时根据指定的Key来识别响应请求的UIAbility实例

  {
    "Contain":"保持宽高比缩放使得图片完全显示在显示边界内",
    "Cover":"默认值,保持宽高比进行缩小或者放大，使得图片两边都大于或等于显示边界 ",
    "Auto":"自适应显示",
    "Fill":"不保持宽高比进行放大缩小，使得图片充满显示边界",
    "ScaleDown":"保持宽高比显示，图片缩小或者保持不变"
  }

组件状态管理

@State //组件内状态管理
@Prop //从父组件单项同步状态
@Link //与父组件双向同步状态
@Watch('onChanged') //监听状态变化 触发回调 
@Provide //跨组件层级双向同步状态  对其所有后代组件可用 可以更新子孙节点数据并出发页面渲染
@Consume //获得提供者组件提供的状态变量的访问
